# 优化会话浏览器 UI 与 Markdown 导出格式

## 背景

Issue #1 已完成了 Kimi 会话浏览器的基础功能。当前页面顶部的图标使用了 Element Plus 自带的聊天图标 `ChatLineRound`，与 Kimi 品牌无关；详情页对问答对仅展示原生文本，无法渲染 Markdown；导出的 Markdown 文件中「用户 / 模型」分节标题使用 `##`，容易与会话内容中的 `##` 标题混在一起，导致大纲结构混乱。

## 目标

对 KimiNoteBook 进行三项体验优化：

1. **品牌 Logo**：将顶部标题栏的聊天图标替换为 Kimi Logo。
2. **Markdown 渲染开关**：在会话详情页增加悬浮切换开关，允许用户在「原生 Markdown 源码」与「渲染后的 Markdown」两种展示模式间切换。
3. **导出大纲降级**：导出 Markdown 中「用户 / 模型」分节标题作为一级标题（`# 用户` / `# 模型`），其内部所有 Markdown 标题统一降级一级（`#` → `##`、`##` → `###`，以此类推），避免用户/模型的大纲融进回答内容的大纲里。

## 待验证的 TDD 验收标准

### 前端 UI

- [ ] 列表页顶部标题左侧展示 Kimi Logo 图标（SVG）。
- [ ] 详情页右下角出现悬浮开关，初始状态为「渲染模式」。
- [ ] 切换开关后，所有问答对内容同步在「源码」与「渲染」之间切换，无需刷新。
- [ ] 渲染模式下，内容中的 Markdown 语法被正确渲染为 HTML。
- [ ] 源码模式下，内容以等宽字体、保留换行和缩进的方式展示。

### 后端 / 导出

- [ ] `POST /api/sessions/{id}/export` 返回的 Markdown 中，用户/模型分节标题为 `# 用户` / `# 模型`。
- [ ] 用户/模型内容中的 Markdown 标题统一降级一级。
- [ ] 导出的文档标题仍为 `# {session.title}`（可选，作为文件顶部标题）。
- [ ] 后端现有测试与新增测试均通过。

## 建议测试命令

```bash

cd server && .venv\Scripts\python.exe -m pytest -v

# 前端测试
cd web && npm test

# 前端构建
cd web && npm run build
```

## 非目标

- 不修改会话解析逻辑。
- 不修改首页列表布局与搜索逻辑。
- 不引入复杂的 Markdown 编辑器，仅做只读渲染。
